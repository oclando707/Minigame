DialogueManager API

# scripts/dialogue_manager.gd (AutoLoad)
extends Node

## 显示对话。lines=字符串数组，textbox_scene=textboxA/textboxB
func show_dialogue(
    lines: Array[String],
    player: CharacterBody2D,
    textbox_scene: String = "res://scene/textboxB.tscn",
    on_finished: Callable = Callable()
) → void

## 当前是否正在显示对话
var is_active: bool # 只读

与原代码的行为对照

原行为 DialogueManager 保持
防止重复打开对话（检查 _dialog 是否为 null） ✅ is_active 守卫，重复调用直接 return
冻结玩家移动（set_movement_enabled(false)） ✅ 参数接收 player 引用，自动冻结/解冻
CanvasLayer + textbox 场景实例化 ✅ 完全相同流程
隐藏 VBoxContainer（选项按钮） ✅ 自动检测并隐藏
按任意键推进文本/关闭 ✅ 完全相同的 _input 逻辑
对话完成清理 CanvasLayer ✅ queue_free + 恢复玩家移动
对话完成后的回调 🆕 on_finished Callable（可选）

textboxA vs textboxB 差异处理

原代码中唯一的区分点：textboxA 和 textboxB 是不同的 UI 资源。
DialogueManager 通过 textbox_scene 参数 让调用方指定，不内置死路径。
